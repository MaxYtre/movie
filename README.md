# 🎬 Movie Scraper (Автоматизированное Планирование и Рекомендации Фильмов)

![Python](https://img.shields.io/badge/python-3.9+-blue.svg)
![License](https://img.shields.io/badge/license-MIT-green.svg)
![GitHub Actions](https://img.shields.io/badge/build-passing-brightgreen.svg)

Данный проект представляет собой автоматизированную систему для сбора информации о зарубежных фильмах, демонстрируемых в кинотеатрах (с фокусом на Пермь), и генерации интерактивного **ICS-календаря**. 

Основная цель — предоставить пользователям удобный способ отслеживать расписание иностранных кинопоказов, обогащенных дополнительными данными (рейтинги, трейлеры, описания).

## ✨ Ключевые особенности

*   **Автоматический сбор данных:** Ежедневный скрейпинг расписаний фильмов с `afisha.ru`.
*   **Фильтрация:** Автоматический отбор только зарубежных фильмов (исключая российские и советские).
*   **Обогащение данных:** Интеграция с внешними API (Kinopoisk.dev, OMDb, YouTube) для получения рейтингов (IMDb, Кинопоиск), оригинальных названий, трейлеров и постеров.
*   **Генерация календаря:** Создание стабильного, идемпотентного ICS-файла, который можно импортировать в любой календарь (Google Calendar, Outlook, Apple Calendar и т.д.).
*   **Веб-интерфейс (отладочный):** Простой HTML-файл для просмотра текущего состояния календаря и отладочной информации.

## 🚀 Установка и запуск (Локально)

Для запуска проекта локально вам понадобится Python версии 3.9 или выше.

1. **Клонируйте репозиторий:**
   ```bash
   git clone https://github.com/your-username/movie.git
   cd movie
   ```

2. **Установите зависимости:**
   Проект использует `pyproject.toml`. Рекомендуется устанавливать зависимости через [Poetry](https://python-poetry.org/) или стандартным `pip`:
   ```bash
   # Через Poetry
   poetry install
   
   # Или через pip
   pip install .
   ```

3. **Настройте переменные окружения:**
   Вы можете скопировать `.env.example` (если есть) или создать свой файл `.env`, чтобы переопределить настройки (например, `MOVIE_SCRAPER_LOG_LEVEL=DEBUG`).

4. **Запуск скрейпера:**
   ```bash
   python -m movie_scraper
   ```
   В результате в папке `docs/` будут сгенерированы актуальные файлы `calendar.ics` и `index.html`.

## ⚙️ Как это работает

Проект функционирует преимущественно в автоматическом режиме через **GitHub Actions**:

1.  **Сбор данных:** Основной процесс инициируется автоматически по расписанию (ежедневно) или вручную. Он включает в себя скрейпинг, парсинг, обогащение (Kinopoisk/IMDb/YouTube) и сохранение в локальную SQLite БД.
2.  **Генерация календаря и отчетности:** После сбора данных система генерирует:
    *   **ICS-файл:** `docs/calendar.ics`
    *   **HTML-индекс:** `docs/index.html`
    *   **Диагностический файл:** `docs/diag.txt`
3.  **Публикация:** Сгенерированные файлы публикуются прямо в репозитории.

**Рабочие процессы GitHub:**
*   [`test-on-pr.yml`](.github/workflows/test-on-pr.yml): Запускает тесты при каждом PR.
*   [`daily-update.yml`](.github/workflows/daily-update.yml): Ежедневное автоматическое обновление.

## 🧠 Алгоритмы Отбора Фильмов и Сеансов

### 1. Отбор Фильмов
*   **Сбор:** Начинается со страницы расписания кинотеатров на `afisha.ru` (до 50 фильмов по умолчанию).
*   **Фильтрация:** Фильмы, произведенные в РФ/СССР, исключаются (проверка свойства `is_foreign`).
*   **Предстоящие сеансы:** Фильм попадает в календарь только если есть хотя бы один сеанс в будущем.

### 2. Обогащение Данных
*   **Рейтинги:** Kinopoisk.dev API для оригинального названия и рейтинга КП. OMDb API для IMDb.
*   **Медиа:** YouTube API для поиска трейлера, парсинг `afisha.ru` для получения постеров.
*   **Цены и даты:** Парсинг цен на билеты со страницы сеансов. Очистка названий для улучшения поиска.

### 3. Формирование Календаря
*   Формат iCalendar (.ics), совместимый со всеми основными календарями.
*   **Идемпотентность:** Гарантирует стабильность обновлений (одинаковый вход = одинаковый выход).
*   **Подавление (30 дней):** Во избежание дублирования, система не публикует в календарь фильм на ту же дату, если он уже был опубликован менее 30 дней назад.

## 🛠 Технические Нюансы

*   **Язык:** Python 3.9+ с использованием `asyncio`.
*   **Парсинг:** `aiohttp` для запросов, `BeautifulSoup` (с `lxml`) для парсинга HTML.
*   **База данных:** `aiosqlite` для хранения (таблицы `films`, `screening_sessions`, `published_events`).
*   **Модели:** Pydantic (`Film`, `ScreeningSession`, `PublishedEvent` и др.).
*   **Устойчивость (Rate Limiting & Retries):** Динамические задержки, механизм `robust_get()` с экспоненциальной задержкой. Кэширование данных в SQLite.
