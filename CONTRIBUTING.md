# Руководство для контрибьюторов (Contributing Guidelines)

Мы рады любому вкладу в развитие проекта Movie Scraper! Ваша помощь делает этот инструмент лучше.

## Как вы можете помочь?

- 🐛 **Сообщить об ошибке**: Если вы нашли баг, пожалуйста, создайте Issue с подробным описанием проблемы и шагами для ее воспроизведения.
- 💡 **Предложить улучшение**: Есть идея для новой фичи? Создайте Issue с меткой `enhancement` и опишите, почему это будет полезно.
- 💻 **Написать код**: Мы всегда рады Pull Request'ам (PR), исправляющим ошибки или добавляющим новый функционал.
- 📖 **Улучшить документацию**: Документация не менее важна, чем код. Если вы видите неточности или знаете, как описать что-то понятнее — смело предлагайте правки!

## Процесс разработки (Pull Requests)

1. **Форкните репозиторий** и создайте новую ветку (`git checkout -b feature/amazing-feature`).
2. **Настройте окружение** локально (мы используем `poetry` для управления зависимостями). Убедитесь, что все зависимости установлены:
   ```bash
   poetry install
   ```
3. **Напишите код**. Старайтесь придерживаться существующих стандартов кодирования. Убедитесь, что ваш код покрыт тестами (если применимо).
4. **Запустите линтеры и тесты**. Перед созданием PR, пожалуйста, убедитесь, что ваш код проходит проверки (в проекте настроены GitHub Actions на PR, но лучше проверить локально).
5. **Создайте Pull Request**. Опишите ваши изменения, используя предложенный шаблон PR. Ссылайтесь на Issue, который решает ваш PR (например, `Fixes #123`).

## Стиль кода

* Проект написан на Python. Мы стараемся следовать рекомендациям PEP 8.
* Используйте аннотации типов там, где это возможно, чтобы улучшить читаемость и надежность.

## Вопросы?

Если у вас есть вопросы по архитектуре или вы не уверены, с чего начать — создайте Issue с тегом `question`, и мы с радостью поможем!

Спасибо за ваш интерес к проекту!
